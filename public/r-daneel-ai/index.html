State-of-the-Art Core Architecture

By splitting strategic planning and logical checking, R_Daneel_AI plays highly positional chess while guaranteeing perfect conformity to rules.

🧠

Modular LLM Strategist

Powered by top-tier models (DeepSeek v4 Pro or Gemma 26B). Evaluates positional themes, coordinates developmental pieces, and identifies tactical motifs through structured reasoning.

🛡️

Prolog Safety Referee

Uses the symbolic SWI-Prolog library Queen as a safety guard. Intercepts LLM coordinates to check legality, calculate pins/checks, and prevent coordinate hallucinations.

⚖️

Tension-Aware Utility

Applies a mathematical -0.5 penalty to captures that resolve board tension on the opponent's terms, steering the LLM toward strategic developing moves instead.

🛡️

Minimax Evasion Guard

If all candidate moves lead to negative utility, the bot runs minimax simulations on all King escape routes to find the path that maximizes resistance and avoids quick mate-in-ones.

📝

Autopsy Log Recording

Generates detailed JSON Lines and CSV logs of the LLM's System 2 thoughts, enabling complete playback of strategic decisions and Prolog referee valuations.

🧬

Open & Extensible

Built with python, PySwip, and vLLM. Seamlessly hook in custom open-weights models, extend Prolog heuristics, or build your own custom rulesets.

Deploy R_Daneel_AI Locally

                
Run your own instance of the bot on Lichess or test locally against Stockfish. The engine is written in Python and leverages a local vLLM server or standard APIs.

            


            

                
1. Install Dependencies

                
Ensure you have SWI-Prolog installed on your system, then install requirements:

                
                

                    

                        Terminal
                        bash
                    

                    
# Install SWI-Prolog first (sudo apt install swi-prolog)
pip install -r requirements.txt
pip install -e ./Queen

                


                
2. Configure Model Backend

                
Create a .env file in the project root to select your engine configurations:


                

                    

                        Config File
                        .env
                    

                    
# Lichess Bot Settings
LICHESS_API_TOKEN=your_lichess_token
LICHESS_MY_USERNAME=R_Daneel_AI

# Model Backend - Choose your LLM brain:
LICHESS_MODEL_NAME=deepseek-v4-pro
# LICHESS_MODEL_NAME=cyankiwi/gemma-4-26B-A4B-it-AWQ-4bit

                


                
3. Start the Bot

                
Run the main Python controller to connect the bot live to your Lichess account:


                

                    

                        Terminal
                        bash
                    

                    
# Spin up the Lichess polling client
python3 lichess_bot.py

                

            

        


        
        

            

                
Technical Q&A

                
Deep-dive details on the engineering decisions, performance metrics, and concurrency limits of the bot.

            


            

                

                    Why use a hybrid Neuro-Symbolic approach instead of pure reinforcement learning?
                    

                        
Pure RL engines like AlphaZero/Lc0 require millions of games and massive compute. On the other hand, raw LLMs have strategic intuition but fail at spatial geometry, frequently outputting illegal moves or coordinate blunders.

                        
R_Daneel_AI uses a symbolic Prolog referee (Queen) to act as the spatial "eyes" for the LLM. Prolog calculates check limits, discoveries, and legal vectors, freeing the LLM to focus entirely on high-level strategic reasoning, development, and positional plans.

                    

                


                

                    How do the DeepSeek v4 and Gemma 26B backends compare in actual play?
                    

                        
Across our 1,000 puzzle benchmark, they ended in an overall tie at 58.1% accuracy. However, their performance curves are highly distinct:

                        
Gemma 26B displays higher consistency in lower-Elo tiers (600–1500), achieving 80.8% accuracy due to simple, robust developmental suggestions.

                        
DeepSeek v4 excels in high-Elo tactical complexity (1600–2500), outperforming Gemma by 2.8% overall in high-Elo brackets (and up to +12% in specific complex tiers like 2200 and 2400) because of its superior deep multi-step reasoning capabilities.

                    

                


                

                    Is the SWI-Prolog engine thread-safe?
                    

                        
No, the PySwip wrapper interface to SWI-Prolog is not thread-safe. Concurrent threads sharing the same Prolog foreign pointer will experience segfaults and garbage collection crashes.

                        
To scale benchmarking or concurrent game handling, R_Daneel_AI avoids multi-threading. Instead, it utilizes independent process sandboxing (multiprocessing) to run isolated Prolog engines, ensuring 100% stability across parallel sessions.

                    

                


                

                    How is the estimated Elo rating calculated in the benchmarks?
                    

                        
We estimate Elo using a bisection-based likelihood solver. Based on the pass rates in each tier, the algorithm solves for the ELO rating that maximizes the probability of obtaining the observed results, assuming a standard logistic player rating curve. This provides a highly accurate reflection of Lichess rating standards.

                    

                

            

        


        
        

            

                
Powered by Queen 👑

                
Meet the underlying engine that makes AI chess reasoning possible. Queen is a Prolog-based chess analyzer equipped with Model Context Protocol (MCP) capabilities.

            

            
            

                
Symbolic Constraints & LLM Collaboration via MCP

                

                    Queen serves as an autonomous chess referee and analyzer. By exposing standard chess diagnostics—like rule validation, pins, discovered attacks, and checks—as MCP tools, any Model Context Protocol-compatible AI agent can hook into Queen to "see" and interact with the chessboard dynamically.
